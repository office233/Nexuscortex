# Portfolio positioning

NexusCortex should be presented as an experimental cognitive-systems prototype, not as a finished frontier AI claim.

## What this repository demonstrates

- Experimental sparse and ternary compute ideas.
- RGBA32-style packed representations for compact state and weights.
- Radio-inspired signal routing experiments.
- Local learning and memory-oriented architecture experiments.
- Go systems implementation, tests, and optional GPU experiments.

## Best-fit roles

This project is strongest for roles involving:

- research engineering;
- experimental AI systems;
- model-efficiency prototypes;
- sparse computation;
- local AI systems;
- Go-based systems prototyping.

## Interview framing

Use this framing:

> I built an experimental neuro-inspired system to explore sparse state, ternary representations, frequency-style routing, and local memory mechanisms. It is a research prototype, and the strongest signal is the ability to translate unusual technical ideas into working code and tests.

## What to avoid saying

Avoid claiming that the project replaces LLMs or proves a new intelligence paradigm. Strong evaluators will respect clear boundaries and reproducible evidence more than broad claims.
