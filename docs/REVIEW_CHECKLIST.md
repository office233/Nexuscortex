# Review checklist

## Before sharing

- Confirm README claims match the current implementation.
- Add a simple architecture diagram.
- Add a short reproducible demo.
- Add baseline comparisons for simple memory and text tasks.
- Add a feature table with current, experimental, and planned items.

## Suggested reviewer flow

1. Read README.
2. Read `docs/PORTFOLIO_POSITIONING.md`.
3. Read `docs/REVIEWER_NOTES.md`.
4. Run tests.
5. Explain the main components and their current limits.

## Interview posture

Present this as an experimental systems project. The strongest signal is technical curiosity, implementation depth, and iteration.
